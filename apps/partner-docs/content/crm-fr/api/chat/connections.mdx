---
title: "GET /api/v1/chat/connections — liste des contacts"
description: "Récupère les utilisateurs avec lesquels vous pouvez chatter dans le portail HeyZack. La forme de la réponse dépend de votre rôle — partenaire, technicien ou administrateur."
---

Ce point de terminaison renvoie la liste des contacts disponibles pour le chat. Aucun paramètre de requête n’est nécessaire — l’API déduit votre rôle à partir du jeton Bearer et renvoie le jeu de contacts adapté. Les partenaires voient leurs techniciens connectés plus l’administrateur. Les techniciens voient leurs partenaires connectés plus l’administrateur. Les administrateurs voient tous les partenaires et techniciens de la plateforme.

## Point de terminaison

```
GET /api/v1/chat/connections
```

## Authentification

Jeton Bearer obligatoire. Voir [Authentification](/api/authentication).

## Paramètres de requête

Aucun. L’identité et le rôle sont dérivés du jeton.

## Exemple

```bash
curl "https://<your-portal-domain>/api/v1/chat/connections" \
  -H "Authorization: Bearer <your-token>"
```

## Format des identifiants de salle (`roomId`)

À l’ouverture d’une conversation, l’identifiant de salle est dérivé de façon déterministe des identifiants des participants. Ce `roomId` apparaît dans les réponses des messages et de l’historique.

| Type de conversation | Format du room ID | Exemple |
|-------------------|---------------|---------|
| Partenaire ↔ Technicien | `room_{id1_trié}_{id2_trié}` | `room_partner_abc123_handyman_xyz456` |
| Admin ↔ Partenaire | `room_admin_{partnerId}` | `room_admin_partner_abc123` |
| Admin ↔ Technicien | `room_admin_{handymanId}` | `room_admin_handyman_xyz456` |

<Note>
  Pour les salles partenaire–technicien, les deux identifiants sont triés alphabétiquement avant concaténation, afin que les deux parties utilisent toujours la même salle quel que soit l’initiateur.
</Note>
